## Current State

The 57-task UI redesign is merged to main. The app loads Casey Central (183 notes) and renders markdown in a rich editor. Core infrastructure (stores, hooks, renderers, design tokens) is solid. Visual polish is in progress.

## What Works

- **3-panel layout**: Sidebar (file tree) | Content (editor/graph/skills) | Terminal (xterm.js)
- **Sidebar**: File tree with directory collapse, item counts, search input, sort modes
- **Editor**: Tiptap rich text with markdown parsing, breadcrumb nav, toolbar, source mode toggle
- **Terminal**: Multi-session tabs, search (Cmd+F), zoom, colored status dots, Claude Code runs inside it
- **Skills panel**: Loads .claude/commands/ from vault, displays skill cards
- **Settings**: Modal with 5 tabs, persistent via localStorage
- **Status bar**: Context-sensitive (editor: cursor/words, graph: nodes/edges)
- **Design system**: CSS custom properties, design tokens, prefers-reduced-motion, thin scrollbars
- **Command palette**: Cmd+K fuzzy search across notes and commands

## What Needs Work

### High Priority (Visual/Functional)

1. **Markdown rendering quality**: Tiptap parses markdown but formatting needs polish to match Obsidian quality (typography, spacing, blockquotes, tables, code blocks). Tables require @tiptap/extension-table.
2. **Graph not rendering**: Shows "No notes yet" because Obsidian files lack TE frontmatter. Need to either: (a) generate graph from wikilinks/file structure, or (b) parse Obsidian-style frontmatter into artifacts.
3. **File saving**: Editor loads files but doesn't write changes back to disk (autosave or manual save needed).
4. **Titlebar**: Shows vault name only. Target has workspace tabs.

### Medium Priority

5. **Editor prose styling**: Need proper CSS for headings, lists, blockquotes, code blocks, links within the Tiptap editor (Tailwind typography plugin or custom styles).
6. **Graph from plain markdown**: Build graph edges from `[[wikilinks]]` and folder structure instead of requiring TE frontmatter.
7. **Sidebar active highlight**: File highlight on select needs more visible styling.
8. **Search**: Sidebar search input is wired but handler is a TODO.

### Low Priority

9. **New file/folder**: Buttons exist but handlers are TODOs.
10. **Vault re-index**: Command palette item exists but no handler.
11. **Graph zoom-to-fit**: Command exists but no handler.
12. **Autosave**: Settings store has autosaveInterval but no implementation.
