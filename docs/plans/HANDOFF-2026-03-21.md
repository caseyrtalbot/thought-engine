## Key Design Decisions (from research + council)

- **Canvas is mission control, not a code editor.** Don't compete with VS Code on text editing.
- **Auto-layout by default.** Every canvas IDE that gave infinite freedom failed (Code Bubbles, Haystack).
- **Keyboard-first navigation.** The highest-value adopters (tmux/vim users) reject mouse-dependent tools.
- **Accumulated value.** Workspace layouts and configurations must persist and compound.
- **The "can't go back" mechanism**: Tools become indispensable by introducing a new mental model users cannot un-think. The model here: "Your work is a living system of parallel contexts with observable relationships."
- **The gap this fills**: No existing tool spatially visualizes multi-agent AI work across files. Warp has a list. This has a map.
